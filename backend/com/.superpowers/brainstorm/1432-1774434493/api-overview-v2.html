Episode & Panel API 设计方案

Panel 负责所有视频生产，Episode 只管 CRUD + 后期

业务流程

Script → Character → Panel（分镜+生产） → Episode（后期合成）

PanelController — 分镜 CRUD + 全部生产逻辑

路径: /api/projects/{projectId}/episodes/{episodeId}/panels

CRUD:
GET /panels — 分镜列表
GET /panels/{panelId} — 分镜详情
POST /panels — 创建分镜
PUT /panels/{panelId} — 更新分镜
DELETE /panels/{panelId} — 删除分镜
分镜生成流程（从 StoryController 迁入）:
POST /panels/generate — AI 生成分镜
POST /panels/confirm — 确认分镜
POST /panels/revise — 修改分镜
POST /panels/retry — 重试生成
单格生产流程（从 EpisodeController 迁入）:
GET /panels/{id}/background — 背景状态
POST /panels/{id}/background — 生成背景
GET /panels/{id}/fusion — 融合状态
POST /panels/{id}/fusion — 生成融合
GET /panels/{id}/transition — 转场状态
POST /panels/{id}/transition — 生成转场
GET /panels/{id}/video-task — 视频任务状态
POST /panels/{id}/produce — 一键生产单格
GET /panels/{id}/production-status — 完整生产状态
GET /panels/{id}/tail-frame — 尾帧信息
批量/集级操作（从 EpisodeController 迁入）:
GET /panels/grid-info — 网格信息
POST /panels/fusion-image — 上传融合图
POST /panels/submit-fusion-page — 提交融合页
POST /panels/split-grid-page — 切分网格
GET /panels/panel-states — 所有面板状态
POST /panels/auto-produce-all — 批量生产
POST /panels/synthesize — 合成最终视频
GET /panels/video-segments — 视频片段列表

EpisodeController — CRUD + 后期合成

路径: /api/projects/{projectId}/episodes

CRUD:
GET /episodes — 剧集列表（分页）
GET /episodes/{episodeId} — 剧集详情
POST /episodes — 创建剧集
PUT /episodes/{episodeId} — 更新剧集
DELETE /episodes/{episodeId} — 删除剧集
后期（预留）:
POST /episodes/{episodeId}/production-status — 管线状态查询
POST /episodes/{episodeId}/retry-production — 重试
（未来 FFmpeg 剪辑拼接接口预留）

废弃

StoryController — 职责拆分完毕后整体删除
原 EpisodeController 的大量生产接口 — 已迁移到 PanelController

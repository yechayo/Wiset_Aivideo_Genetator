Episode & Panel API 架构总览

基于前端现有交互逻辑，重新设计 Episode（剧集）和 Panel（分镜）接口

设计原则

  • Panel = 生产驱动者 — 所有视频生产操作（背景、融合、转场、视频生成）全部挂在分镜层
  • Episode = 容器 + 后期 — 管理分镜集合，所有分镜视频生成完后做剪辑/拼接
  • 路径统一 — 嵌套在 /api/projects/{projectId} 下，与 CharacterController 一致
  • 前端兼容 — 保留前端已有的调用模式，只调整路径前缀
业务流程 & API 归属
ScriptController
剧本
→
CharacterController
角色
→
PanelController
分镜 + 生产
→
EpisodeController
剧集 + 后期
PanelController — 所有生产逻辑
GET /panels — 分镜列表
GET /panels/{panelId} — 分镜详情
POST /panels/generate — AI生成分镜
POST /panels/confirm — 确认分镜
POST /panels/revise — 修改分镜
POST /panels/retry — 重试生成
—— 单格生产流程 ——
POST /panels/{id}/background — 生成背景
POST /panels/{id}/fusion — 生成融合
POST /panels/{id}/transition — 生成转场
POST /panels/{id}/produce — 一键生产
POST /panels/auto-produce-all — 批量生产
POST /panels/synthesize — 合成最终视频
EpisodeController — CRUD + 后期合成
GET /episodes — 剧集列表（分页）
GET /episodes/{id} — 剧集详情
POST /episodes — 创建剧集
PUT /episodes/{id} — 更新剧集
DELETE /episodes/{id} — 删除剧集
（后期 FFmpeg 剪辑拼接预留）

关键变化点

从 EpisodeController 移出

  • 所有 panel 级别的生产接口
  • 融合图上传、提交融合页
  • 网格切分、网格信息
  • 单格操作（背景/融合/转场/视频）
  • auto-produce-all、synthesize

从 StoryController 移入

  • 分镜生成 (generate)
  • 分镜确认 (confirm)
  • 分镜修改 (revise)
  • 分镜重试 (retry)
  • StoryController 整体废弃

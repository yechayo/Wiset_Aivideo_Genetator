设计总结

Project Controller 重构 — 完整设计方案

核心决策

实体结构
Project / Episode / Character 统一 Map，重写所有 Service
逻辑删除
Project 新增 deleted 字段，Repository 自动过滤
剧本大纲
AI 输出扁平 JSON（characters + items + episodes），存 projectInfo["script"]
剧集生成
确认大纲后自动分批生成，用户无感知章节概念
状态回退
advance 支持 backward，精确清理对应阶段数据
Controller 拆分
ProjectController = CRUD + 状态，ScriptController = 剧本操作

4 阶段实施计划

1
Entity + deleted + Repository
Project 新增 deleted 字段，ProjectRepository 所有查询加 deleted=false 过滤，新增分页查询方法。定义 projectInfo / episodeInfo / characterInfo 的 key 常量。
2
ProjectStatus 枚举 + DTO 更新
枚举新增 getPreviousStatus/getNextStatus/canGoBack/canAdvance/getStepHistory。ProjectStatusResponse 新增 5 个字段。新增 AdvanceRequest、ProjectUpdateRequest、StepHistoryItem DTO。
3
Service 层重写（核心）
PipelineService / ScriptService / CharacterExtractService / CharacterImageGenerationService / StoryboardService / EpisodeProductionService / EpisodeController — 全部从字段访问改为 Map 访问。PromptBuilder 大纲生成改为输出结构化 JSON。ScriptService 确认逻辑改为自动分批生成。
4
Controller 重构
重构 ProjectController（CRUD + 分页 + 状态 + 回退）。新建 ScriptController（5 个接口）。删除 ProjectController 中的脚本相关接口。

不变更的组件

StoryController、FileController、AuthController、ConfigController、JobController、TaskController — 保持不变。

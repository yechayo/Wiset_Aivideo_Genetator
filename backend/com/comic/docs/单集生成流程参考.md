## 第三步：分镜脚本生成

**做什么**：把一集剧本拆解成一个个独立的镜头（分镜），每个分镜包含详细的拍摄指令。

**输入**：

- 一集的完整剧本文本
- 目标总时长（如60秒）
- 视觉风格（如"仙侠3D动画，半写实风格"）

**AI怎么拆的**：

- 系统会算出最少需要多少个分镜（60秒 ÷ 3秒每镜 = 至少20个）
- 推荐按每镜2.5秒拆（60秒 → 推荐24个分镜）
- 每个分镜时长限制在1到4秒之间

**每个分镜包含的信息**：

| 字段     | 内容         | 示例                                                       |
| -------- | ------------ | ---------------------------------------------------------- |
| 编号     | 第几个镜头   | 第1镜                                                      |
| 时长     | 这个镜头几秒 | 2.5秒                                                      |
| 场景     | 在哪拍的     | 教室，白天，靠窗最后一排                                   |
| 出场角色 | 有谁         | 林霄、苏婉                                                 |
| 景别     | 远近         | 特写                                                       |
| 拍摄角度 | 仰拍俯拍     | 低位仰拍                                                   |
| 运镜方式 | 镜头怎么动   | 缓慢推进                                                   |
| 画面描述 | 具体画面内容 | 林霄坐在靠窗座位上，单手托腮，侧身望向窗外，阳光洒在他脸上 |
| 对白     | 这段说了什么 | "我再也不想当好人了"                                       |
| 视觉特效 | 后期效果     | 浅景深，背景虚化，暖色调                                   |
| 音效     | 声音         | 环境音，教室嘈杂声                                         |

**AI的策略**：

- 高潮/冲突场景 → 用大量1到2秒的短镜头，快速切换营造紧张感
- 对话/情感戏 → 用2到3秒的中等镜头，展现表情细节
- 环境空镜/转场 → 用3到4秒的长镜头，建立空间感
- 动作戏 → 用1秒极短镜头，快速剪辑强化动感

**颗粒度**：每集生成一批分镜，就是这一集专用的。

------

## 第四步：九宫格分镜图生成

**做什么**：把分镜脚本中的画面描述，批量生成成一张张九宫格图片。一张九宫格里包含9个分镜画面。

**怎么组织的**：

- 假设一集有27个分镜，每页九宫格放9个，就需要生成3页
- 3页是**并行同时生成**的，不串行等

**每张九宫格图长这样**：



```
┌─────────┬─────────┬─────────┐
│ 分镜1   │ 分镜2   │ 分镜3   │
│ 教室远景│ 林霄特写│ 苏婉近景│
├─────────┼─────────┼─────────┤
│ 分镜4   │ 分镜5   │ 分镜6   │
│ 操场全景│ 对话中景│ 手部特写│
├─────────┼─────────┼─────────┤
│ 分镜7   │ 分镜8   │ 分镜9   │
│ 建筑远景│ 林霄背影│ 夕阳空镜│
└─────────┴─────────┴─────────┘
```

**AI生成时的提示词构成**（这是最关键的，决定了画面质量）：

**第一部分：布局规范**

- 告诉AI输出3行3列的九宫格
- 每格比例16比9（横屏）
- 格子之间用4像素黑色细线分隔
- 整体分辨率如2K高清

**第二部分：角色一致性约束**

- 把第二步生成的角色三视图/表情包作为参考图喂给AI
- 用代号替换角色中文名（"林霄"变成"角色A"），防止AI乱写文字
- 强制要求：所有面板里的角色必须长得和参考图一模一样，包括脸型、发型、衣服、肤色
- 明确标注"这是不可商量的硬性要求"

**第三部分：场景一致性约束**

- 系统自动分析哪些分镜在同一个场景（比如分镜1、4、7都在教室）
- 生成约束文本：场景1出现在第1、4、7格，要求这三格的教室环境、光线、色调必须完全一致
- 就像"同一个场景用不同机位拍摄"

**第四部分：负面约束**

- 绝对禁止出现任何文字、数字、字母、字幕、水印、对话框气泡、面板编号
- 唯一例外：场景中实物上的文字（如路牌、书封面、横幅）

**第五部分：逐格画面描述**

- 每个格子单独一段，用分隔线隔开
- 每段包含：景别（中文自动翻译成英文专业术语）+ 角度 + 运镜 + 场景 + 画面描述
- 示例："16比9横屏，中景，视平角度，固定镜头。环境：教室。角色A站在窗边，凝视窗外。"

**第六部分：风格描述**

- 从上游节点自动获取的视觉风格
- 如"仙侠3D动画角色，半写实风格，电影级光影，虚幻引擎5渲染质感"

**颗粒度**：每页九宫格是一张完整图片，一次AI调用生成一页。27个分镜 = 3页 = 3次AI图像调用（并行）。

------

## 第五步：九宫格拆解

**做什么**：把一张九宫格大图切成9张独立的小图片。

**怎么做的**：

- 完全在浏览器本地用画布技术完成，不调任何AI
- 加载九宫格图片，计算每格的像素位置
- 九宫格就是3行3列，把图等分成9份
- 每份裁剪出来转成独立的图片数据

**拆解过程**：



```
原图 (6144 × 3456 像素)
    ↓ 等分3×3
每格 2048 × 1152 像素
    ↓ 裁剪出9张
分镜1.png、分镜2.png、分镜3.png ... 分镜9.png
```

**同时做的事**：

- 每张切割出来的小图，自动关联上第三步生成的对应分镜描述数据
- 这样每张图不仅有画面，还附带"这是什么场景、什么景别、什么运镜、什么对白"等全部元数据

**颗粒度**：3页九宫格 → 27张独立分镜图，每张携带完整元数据。

------

## 第六步：视频提示词构建与分组

**做什么**：把分镜图和描述组织成视频AI能理解的格式，然后按时长分组。

**分组逻辑**：

- 用户选择一个最大时长（5秒、10秒或15秒）
- 系统用贪心算法把分镜塞进一个个组里
- 比如最大10秒一组，每个分镜3秒，那每组塞3个分镜（9秒），或者2个（6秒）
- 27个分镜大概分成10个任务组

**提示词构建**（分两种情况）：

**情况一：Sora平台**（不调AI，直接拼接）

- 每组开头自动加一个0.5秒的纯黑空镜（作为视频起手过渡）
- 然后逐个拼接分镜信息，用中文直接输出
- 格式："第N镜 / 时长X秒 / 场景：景别，角度，运镜，画面描述，对白，特效，音效"
- 不做任何翻译和美化，原汁原味给Sora

**情况二：其他平台**（Luma、Runway等，需要调AI翻译）

- 把中文分镜信息发给AI大模型
- AI翻译成英文视频提示词，加上风格前缀
- 输出格式："风格描述 / 第1镜：时长，英文场景描述 / 第2镜..."

**颗粒度**：每个任务组对应一个短视频（6到10秒左右）。

------

## 第七步：图片融合（可选）

**做什么**：把一个任务组里的2到3张分镜图拼成一张参考图，上传后喂给视频AI。

**怎么做的**：

- 在浏览器本地用画布拼接
- 支持三种布局：网格排列、横向一排、竖向一列
- 每张图左上角标序号（黑色半透明底+白色数字）
- 底部加标题栏写"分镜融合图 共N个镜头"
- 如果有角色三视图，也会拼进去
- 拼好后上传到云存储获取网络链接

**为什么需要这一步**：视频AI（如Sora）支持参考图输入，融合图能让AI理解"这几张图是一个连续场景，角色要长得一样"。

**颗粒度**：每个任务组一张融合图。

------

## 第八步：视频生成

**做什么**：把提示词和参考图提交给视频AI平台，生成短视频片段。

**怎么做的**：

- 对每个任务组，调用视频生成接口（如Sora API）
- 提交参数：提示词文本 + 参考图链接 + 画面比例(16比9) + 时长 + 是否高清
- 提交后每隔5秒轮询一次状态
- 生成成功后获取视频下载链接
- 最多等10分钟，超时报错

**颗粒度**：每个任务组生成一个短视频片段。27个分镜分成10组 = 最终10个视频片段。

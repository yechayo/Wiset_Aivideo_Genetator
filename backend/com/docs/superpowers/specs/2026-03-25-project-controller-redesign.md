# Project Controller Redesign

## Background

Current `ProjectController` mixes project CRUD, script management, and pipeline orchestration in a single controller with 12+ endpoints. This redesign separates concerns using strict REST sub-resource paths.

## Design Decisions

1. **ProjectController** only handles project CRUD and status/pipeline
2. **ScriptController** handles all script-related operations (moved out of ProjectController)
3. **No chapter concept** — outline confirmation triggers automatic episode generation
4. **Logical delete** with `deleted` field, queries auto-filter
5. **Status + Pipeline merged** — single status view includes step history, previous/next status, and rollback capability
6. **Confirmed steps are immutable** — going back to a previous stage makes confirmed data read-only

## API Design

### ProjectController — `/api/projects`

#### POST `/api/projects` — Create Project

**Request:**
```json
{
  "storyPrompt": "string",
  "genre": "string",
  "targetAudience": "string",
  "totalEpisodes": 10,
  "episodeDuration": 60,
  "visualStyle": "ANIME"
}
```

**Response:** `{ "data": { "projectId": "PROJ-xxxx" } }`

#### GET `/api/projects` — List Projects

**Query params:** `page`, `size`, `status`, `sortBy` (createdAt|updatedAt), `sortOrder` (asc|desc)

**Response:** paginated list of `ProjectListItemResponse`

#### GET `/api/projects/{projectId}` — Project Detail

Returns full project entity.

#### PUT `/api/projects/{projectId}` — Update Project

Full update of project basic info (storyPrompt, genre, targetAudience, totalEpisodes, episodeDuration, visualStyle).

#### PATCH `/api/projects/{projectId}` — Partial Update

Partial update (e.g., only update name).

#### DELETE `/api/projects/{projectId}` — Logical Delete

Sets `deleted = true`. Auto-filtered from all queries.

#### GET `/api/projects/{projectId}/status` — Project Status

Returns unified status view including step history, navigation, and rollback info.

**Response:**
```json
{
  "projectId": "PROJ-xxx",
  "currentStep": 4,
  "currentStatus": "CHARACTER_REVIEW",
  "previousStatus": "SCRIPT_CONFIRMED",
  "nextStatus": "CHARACTER_CONFIRMED",
  "canGoBack": true,
  "canAdvance": true,
  "isGenerating": false,
  "isFailed": false,
  "isReview": true,
  "completedSteps": [1, 2, 3],
  "stepHistory": [
    { "step": 1, "status": "OUTLINE_GENERATED", "label": "剧本大纲", "confirmed": true },
    { "step": 2, "status": "SCRIPT_CONFIRMED", "label": "剧本确认", "confirmed": true },
    { "step": 3, "status": "CHARACTER_EXTRACTED", "label": "角色提取", "confirmed": true },
    { "step": 4, "status": "CHARACTER_REVIEW", "label": "角色审核", "confirmed": false }
  ],
  "productionProgress": null,
  "storyboardProgress": null
}
```

Enriched data for producing/storyboard phases is included when applicable (same as current `ProjectStatusResponse`).

#### POST `/api/projects/{projectId}/status/advance` — Advance/Rollback Pipeline

**Request (forward):**
```json
{ "direction": "forward", "event": "characters_confirmed" }
```

**Request (backward):**
```json
{ "direction": "backward" }
```

**Rollback rules:**
- `confirmed: true` steps: corresponding data (e.g., confirmed script) becomes read-only
- Rollback clears data generated by subsequent steps (e.g., rolling back to script stage clears generated episodes)

### ScriptController — `/api/projects/{projectId}/script`

All script operations moved from ProjectController. No chapter concept.

#### GET `/api/projects/{projectId}/script` — Get Script Content

Returns outline and all episodes.

#### POST `/api/projects/{projectId}/script/generate` — Generate Outline

Generates script outline via AI.

#### POST `/api/projects/{projectId}/script/confirm` — Confirm Script

Confirms outline and automatically generates all episodes.

#### POST `/api/projects/{projectId}/script/revise` — Revise Script

**Request:**
```json
{
  "revisionNote": "string",
  "currentOutline": "string (optional)"
}
```

#### PATCH `/api/projects/{projectId}/script/outline` — Save Outline

Direct save of user-edited outline without AI regeneration. Clears existing episodes.

**Request:**
```json
{ "outline": "string" }
```

## Unchanged Controllers

- **EpisodeController** (`/api/episodes`) — video production, panel operations
- **StoryController** (`/api/story`) — storyboard generation and review
- **CharacterController**, **FileController**, etc.

## Entity Changes

- `Project` entity: add `deleted` field (Boolean, default false)
- `ProjectRepository`: all queries add `WHERE deleted = false` condition

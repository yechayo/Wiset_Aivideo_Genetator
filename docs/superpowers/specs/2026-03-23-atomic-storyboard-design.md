# Step5 原子化卡片式分镜设计

## 目标

将 Step5 从"分镜审查页"升级为"原子化卡片工作台"，每个分镜格子从确认分镜到生成视频的全流程都在卡片上完成。

## 核心原则

- **从 Step5 开始原子化**：用户确认分镜后，立即进入卡片视图，后续所有操作（场景图、融合、视频）都在卡片上完成
- **流水线 + 原子化并存**：一键自动执行 和 逐格手动操作可以混合使用
- **后端零改动**：复用现有 API，新增接口仅用于场景图重生成

```
     │            │              │              │
     └────────────┴──────────────┴──────────────┘
                         │
                    渲染对应卡片视图
```

### 2.2 阶段1卡片 — 分镜审查态

```
┌─────────────────────────────────────┐
│ #1  中景 / 平视                      │
│                                      │
│ 场景: 明亮的办公室，落地窗透光...      │
```

## 三、融合设计

### 3.1 融合前置条件

融合必须等待该格子对应的场景图生成完成。未生成时，融合按钮禁用。

**状态处理：**
- 场景图生成中：融合按钮禁用，显示等待文案
- 场景图生成失败：该格显示错误标记，融合按钮显示"重新生成场景图"而非融合操作，用户需先重生成场景图

### 3.2 自动融合

**逻辑**：按分镜顺序 + 分镜 JSON 中的角色列表，自动把角色参考图叠加到对应格子。

**流程**：
1. 调用 `splitGridPage(episodeId, pageIndex)` 获取该页切图坐标
2. 根据分镜 index 找到该格子对应的角色列表
3. 获取角色参考图 URL
4. Canvas 按固定坐标叠加角色图
5. 导出 PNG → 上传 OSS → 调用 `submitFusionPageWithAuto(episodeId, pageIndex, urls, false)`

**特点**：非随机，完全按分镜 JSON 的角色绑定关系。

**多角色叠加规则**：多个角色按分镜 JSON 中 `characters[]` 数组的顺序依次叠加（数组靠前的在下，靠后的在上），blend 模式使用 alpha 混合。

### 3.3 手动融合

**逻辑**：打开九宫格模态框，用户拖拽调整角色位置。

**流程**：
1. 点击"手动融合" → 打开 GridFusionEditor 模态框
2. 模态内显示整页九宫格（该格子所属页）
3. 用户拖拽角色到每个格子，调整位置和大小
4. 点击"确认" → 导出 → 上传 → `submitFusionPageWithAuto(..., false)`
5. 模态关闭，卡片更新

**模态取消处理**：用户点击遮罩或关闭按钮时，模态直接关闭，不触发任何后端调用，融合状态保持不变。

### 3.4 融合状态显示

- 未融合：显示场景网格图 + [自动融合] [手动融合]
- 融合中：该格显示 spinner
- 已融合：显示融合图 + [重新融合]

## 四、场景图重生成设计

### 4.1 需求

每个格子的场景图生成后，可以单独重新生成。

### 4.2 需要新增的后端接口

```
POST /api/episodes/{episodeId}/panels/{panelIndex}/regenerate-scene
```

功能：按该分镜的 scene_description 重新生成单格场景图，覆盖 `sceneGridUrls[panelIndex]`。

### 4.3 流程

1. 用户点击"重新生成场景图"
2. 前端调用 `regenerateSceneImage(episodeId, panelIndex)`
3. 后端调用图像生成服务，替换该格子的场景图 URL
4. 前端轮询更新卡片显示
5. 如果该格子已有融合图 → 触发重新融合流程

## 七、技术实现

### 7.1 状态类型

```typescript
type WorkflowPhase = 'review' | 'scene-generating' | 'fusion' | 'video' | 'completed';

interface PanelCardState {
  panelIndex: number;
  // 分镜信息
  shotType: string;
  sceneDescription: string;
  characters: string;
  dialogue: string;
  // 场景图状态
```

## 九、风险与缓解

| 风险 | 缓解 |
|------|------|
| 并发生成多个视频触发平台限流 | 后端已有信号量控制；前端可加节流 |
| 单格失败不影响其他格子 | 每格独立 try-catch，失败可重试 |
| 场景图重生成后融合状态需同步 | 重生成场景图后自动触发重新融合 |
| 九宫格分页与单格索引映射错位 | `panelIndex → pageIndex` 按行优先计算统一封装 |

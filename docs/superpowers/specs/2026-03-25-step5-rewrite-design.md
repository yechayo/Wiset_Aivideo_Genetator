## Step5 — 视频生产

### 页面布局

**展开式列表视图**，按章节分组展示所有剧集。

#### 顶部栏

- 左侧：步骤进度条（Steps 1-6）
- 右侧：已生成/未生成统计 + 预留"一键生成"按钮（后期实现）

#### 主体区域

按章节分组，每集一张可展开的卡片。

### 剧集卡片

**三种状态：**

1. **已完成** — 绿色边框，折叠状态，显示完成统计
2. **进行中** — 白色边框，展开状态，显示片段子卡片列表
3. **未开始** — 灰色边框，折叠状态，可点击展开

**折叠状态内容：**
- 左侧：状态图标 + 集数标题 + 剧情简介
- 右侧：片段完成指示器（每个片段一个小色块：绿=完成，黄=待审核，灰=未开始）+ 展开箭头

**展开状态内容：**
- 卡片 header 同折叠状态
- 下方显示该集所有片段子卡片

### 片段子卡片

**两种状态：**

1. **折叠** — 单行显示简介
2. **展开** — 显示完整工作区

#### 折叠状态

```
[状态图标] 片段 N · 状态    |  场景[图标]  角色[头像×N]  ●—●—●  [展开]
           一行剧情摘要...      |
```

- 左侧：状态图标 + 片段编号 + 状态标签 + 一行剧情摘要
- 右侧：
  - 场景缩略图标
  - 使用的角色头像列表（圆形）
  - 三步进度指示器（场景 → 四宫格 → 视频，绿=完成，黄=当前，灰=未开始）
    > 注意：流水线共 4 步，但步骤 ①（调用角色资产）完全自动、对用户不可见，因此进度指示器只显示 3 格，分别映射：场景（步骤②）、四宫格（步骤③）、视频（步骤④）。
  - 展开箭头

#### 展开状态

**Header** 同折叠状态。

**内容区** 左右 50% 分栏：

- **左侧 — 四宫格漫画**（16:9 横屏比例）
  - 2×2 网格展示四个画面
  - 底部操作区：
    - 审核通过按钮（白底黑字）
    - 重新生成按钮（黄色边框）
  - 修改建议输入框（textarea）
  - 未生成时显示占位

- **右侧 — AI 视频**（16:9 横屏比例）
  - 未开始时：禁用状态，显示"生成视频"按钮（disabled）
  - 审核通过前：占位提示"四宫格审核通过后可生成"
  - 生成中：loading 状态 + 进度提示
  - 审核通过后：视频播放器 + "生成视频"按钮
  - 生成失败：错误提示 + 重试按钮 + 修改建议输入框（同四宫格逻辑）

### 每个片段的生产流水线

4 个步骤，用户手动推进：

```
① 调用角色资产（自动） → ② 生成场景图（自动） → ③ 四宫格漫画（审核点） → ④ 生成 AI 视频（手动触发）
```

- **步骤 ① 调用角色资产**：从 Step4 生成的角色素材中**自动匹配**该片段涉及的角色，同时允许用户**手动修改**（添加/移除/替换角色资产）
- **步骤 ② 生成场景图**：根据分镜描述自动生成场景背景图
- **步骤 ③ 四宫格漫画**：场景图 + 角色资产 + 剧情描述 → AI 生成四宫格漫画
  - **唯一审核点**：用户必须审核通过才能继续
  - 不通过时：用户输入修改建议（类似 prompt），重新生成
  - 失败时：自动重试（最多 3 次），重试耗尽后显示修改建议输入框
- **步骤 ④ 生成 AI 视频**：四宫格漫画 + 剧情描述 → AI 生成 prompt → 生成视频
  - 四宫格审核通过后才能触发
  - 失败时同样自动重试 + 用户可提供修改建议
  - 四宫格审核通过后才能触发

### 步骤衔接逻辑

- 场景图和四宫格绑定为一个阶段：场景图 + 角色资产 → 四宫格
- 四宫格审核通过 → 解锁视频生成
- 不自动串行，每步需要用户手动触发（预留"一键生成"后期实现）

## 数据模型

```
Chapter (章节) → Episode (剧集) → Segment (片段) → Panel (面板/分镜格)
```

- **Chapter**：章节，包含多集
- **Episode**：一集，包含多个片段
- **Segment**：片段，生产流水线的基本单位。每个片段包含 4 个面板
- **Panel**：面板/分镜格，对应剧本中的单个分镜

**片段与四宫格的关系：**
- 每个片段包含 4 个面板
- 4 个面板通过 AI 合成为一张四宫格漫画
- 一张四宫格漫画生成一个 AI 视频片段

**与现有数据模型的关系：**
- 现有 `StoryboardPanel` 对应新的 Panel
- 现有按 episode 管理的 `PanelState` 需要调整为按 segment 管理
- 新增 Segment 层级，每个 Segment 聚合 4 个 Panel 的状态

## 状态管理

- **数据加载**：进入 Step5 时一次性加载所有章节/剧集/片段数据
- **轮询策略**：按 segment 级别轮询状态，避免频繁全局刷新
- **展开状态**：本地 useState 管理，不需要持久化
- **后端 API 适配**：需要在实现阶段确认现有 episode/panel 级 API 如何映射到新的 chapter/episode/segment 模型

## 设计规范

### 视觉风格

- 暗色主题，延续项目现有设计语言
- 字体：Space Grotesk（标题）、Manrope（正文）、Inter（标签）
- 颜色系统：
  - 绿色 `#4ade80` — 已完成
  - 黄色 `#fbbf24` — 待审核/进行中
  - 灰色 `#474747` — 未开始
  - 白色 `#ffffff` — 当前激活

### 交互规范

- 所有卡片支持展开/折叠
- 片段子卡片折叠时只显示单行状态
- 片段子卡片展开时左右 50% 分栏
- 图片比例统一 16:9 横屏
- 角色头像用圆形展示
